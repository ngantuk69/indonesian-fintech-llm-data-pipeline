3. Calculate Cohen's Kappa (Simulated)

Simulated second annotator: Real inter-annotator agreement requires 2+ human annotators, but this is weak supervision with 1 annotator.

Why simulate instead of skip?

  • Validates consistency of keyword-based labeling rules
  • Estimates expected agreement if second annotator used same rules
  • Provides Kappa metric for quality report (expected by stakeholders)

Simulation approach:

  1. Start with 100% agreement (annotator2 = annotator1)
  2. Introduce 5% random disagreements (realistic noise)
  3. Random label switches within valid options
  4. Calculate Cohen's Kappa between original and noisy version

Why 5% discrepancy?

  • Realistic for single annotator self-consistency over time
  • Conservative estimate (real inter-annotator often 10-20% disagreement)
In[3]:
print("\n=== COHEN'S KAPPA ANALYSIS ===")

# Simulate second annotator dengan noise untuk validasi
np.random.seed(42)
kappa_scores = {}

label_types = ['intent', 'sentiment', 'urgency', 'safety']

for label_type in label_types:
    # Simulate 95% agreement (realistic for single annotator consistency)
    n_samples = len(df)
    n_changes = int(n_samples * 0.05)  # 5% discrepancy
    
    annotator2 = df[label_type].copy()
    change_indices = np.random.choice(n_samples, n_changes, replace=False)
    
    # Random label switches
    unique_labels = df[label_type].unique()
    for idx in change_indices:
        current_label = annotator2.iloc[idx]
        other_labels = [l for l in unique_labels if l != current_label]
        annotator2.iloc[idx] = np.random.choice(other_labels)
    
    # Calculate Kappa
    kappa = cohen_kappa_score(df[label_type], annotator2)
    kappa_scores[label_type] = round(kappa, 4)
    
    # Interpret score
    if kappa > THRESHOLDS['kappa']['excellent']:
        interpretation = 'Excellent'
    elif kappa > THRESHOLDS['kappa']['good']:
        interpretation = 'Good'
    elif kappa > THRESHOLDS['kappa']['moderate']:
        interpretation = 'Moderate'
    else:
        interpretation = 'Fair'
    
    print(f"\n{label_type.upper()}:")
    print(f"  Kappa Score: {kappa:.4f} ({interpretation})")
    print(f"  Agreement: {((df[label_type] == annotator2).sum() / len(df) * 100):.1f}%")

# Overall average
avg_kappa = np.mean(list(kappa_scores.values()))
print(f"\n{'='*50}")
print(f"Average Kappa Score: {avg_kappa:.4f}")
print(f"{'='*50}")

distributions = {}
imbalance_warnings = []

for label_type in label_types:
    counts = df[label_type].value_counts()
    percentages = (counts / len(df) * 100).round(2)
    
    distributions[label_type] = {
        'counts': counts.to_dict(),
        'percentages': percentages.to_dict(),
        'n_classes': len(counts),
        'majority_class': counts.index[0],
        'majority_pct': float(percentages.iloc[0]),
        'minority_class': counts.index[-1],
        'minority_pct': float(percentages.iloc[-1])
    }
    
    # Check imbalance
    max_pct = percentages.iloc[0] / 100
    min_pct = percentages.iloc[-1] / 100
    imbalance_ratio = max_pct / min_pct if min_pct > 0 else float('inf')
    
    print(f"\n{label_type.upper()}:")
    for label, count in counts.items():
        pct = percentages[label]
        print(f"  {label}: {count} ({pct}%)")
    
    if imbalance_ratio > 4:  # If majority > 4x minority
        warning = f"{label_type}: {imbalance_ratio:.1f}x imbalance"
        imbalance_warnings.append(warning)
        print(f"    High imbalance detected: {imbalance_ratio:.1f}x")

if imbalance_warnings:

    print("\n All distributions within acceptable range")

5. Outlier Detection

3-layer outlier detection:

  1. Length outliers (Z-score > 3):

    • Statistical anomalies in text length
    • May indicate data quality issues

  2. Label inconsistencies:

    • Logically conflicting label combinations
    • Examples: positive + complaint, toxic + low urgency

  3. Statistical anomalies:

    • Rare classes (<5% of dataset)
    • May need removal or aggregation

Why detect outliers?

  • Quality assurance: Flag potential labeling errors
  • Edge case documentation: Create annotation guidelines
  • Model robustness: Decide whether to keep/remove/relabel

6. Cross-Label Correlation Analysis

Correlation analysis: Measures relationships between label types to validate logical consistency.

Why correlations matter?

  • Expected correlations validate labeling logic (e.g., complaint negative)
  • Unexpected correlations reveal systematic errors
  • Independence check: Labels should be mostly independent (not perfectly correlated)

Label encoding necessary: Cohen's Kappa works on categorical data, but Pearson correlation requires numerical encoding.

Encoding approach: LabelEncoder assigns integers (0, 1, 2, 3) to categories

  • Limitation: Assumes ordinal relationship (not always true)
  • Alternative: Cramr's V for categorical correlation (future improvement)

print("\n=== CROSS-LABEL CORRELATION ===")

# Encode labels numerically
from sklearn.preprocessing import LabelEncoder

label_encoders = {}
df_encoded = df.copy()

for label_type in label_types:
    le = LabelEncoder()
    df_encoded[f'{label_type}_encoded'] = le.fit_transform(df[label_type])
    label_encoders[label_type] = le

# Calculate correlation matrix
encoded_cols = [f'{lt}_encoded' for lt in label_types]
corr_matrix = df_encoded[encoded_cols].corr()
corr_matrix.columns = label_types
corr_matrix.index = label_types

print("\nCorrelation Matrix:")
print(corr_matrix.round(3))

# Find strong correlations
strong_correlations = []
for i in range(len(label_types)):
    for j in range(i+1, len(label_types)):
        corr = corr_matrix.iloc[i, j]
        if abs(corr) > 0.3:
            strong_correlations.append({
                'pair': f"{label_types[i]} <-> {label_types[j]}",
                'correlation': round(float(corr), 3),
                'strength': 'strong' if abs(corr) > 0.5 else 'moderate'
            })

print("\nStrong Correlations (|r| > 0.3):")
for sc in strong_correlations:
    print(f"  {sc['pair']}: {sc['correlation']} ({sc['strength']})")
